2. TYPOGRAPHY - HỆ THỐNG FONT CHỮ
================================================================================

2.1. FONT FAMILIES
------------------
Primary Font: Inter, system-ui, -apple-system, sans-serif
- Sử dụng cho toàn bộ text trong ứng dụng
- Font chữ hiện đại, dễ đọc, responsive tốt

2.2. FONT SIZES
---------------
• text-xs:   0.75rem (12px)  - Captions, footnotes, labels nhỏ
• text-sm:   0.875rem (14px) - Body text phụ, descriptions
• text-base: 1rem (16px)     - Body text chính (DEFAULT)
• text-lg:   1.125rem (18px) - Subheadings, emphasized text
• text-xl:   1.25rem (20px)  - Section headings
• text-2xl:  1.5rem (24px)   - Card titles, page subtitles
• text-3xl:  1.875rem (30px) - Page titles
• text-4xl:  2.25rem (36px)  - Hero headings, main titles
• text-5xl:  3rem (48px)     - Landing page heroes
• text-6xl:  3.75rem (60px)  - Extra large display text

2.3. FONT WEIGHTS
-----------------
• font-light:     300 - Subtle text, quotes
• font-normal:    400 - Body text (DEFAULT)
• font-medium:    500 - Emphasis, button labels
• font-semibold:  600 - Subheadings, card titles
• font-bold:      700 - Headings, important labels
• font-extrabold: 800 - Display text, heroes

2.4. LINE HEIGHTS
-----------------
• leading-none:    1     - Tight headings
• leading-tight:   1.25  - Compact text
• leading-normal:  1.5   - Body text (DEFAULT)
• leading-relaxed: 1.625 - Comfortable reading
• leading-loose:   2     - Spacious paragraphs

2.5. LETTER SPACING
-------------------
• tracking-tight:  -0.025em - Headings
• tracking-normal: 0        - Body text (DEFAULT)
• tracking-wide:   0.025em  - Buttons, labels
• tracking-wider:  0.05em   - All-caps text

11. ACCESSIBILITY STANDARDS - QUY CHUẨN TRUY CẬP
================================================================================

11.1. COLOR CONTRAST
--------------------
• Text on background: Minimum 4.5:1 ratio
• Large text (18px+): Minimum 3:1 ratio
• Primary blue on white: 7:1 (excellent)
• Gray-700 on white: 5.5:1 (good)

11.2. FOCUS INDICATORS
----------------------
• Always visible focus ring: ring-2 ring-primary-500
• Never remove focus outline without replacement
• Keyboard navigation must be fully supported

11.3. SEMANTIC HTML
-------------------
• Use proper heading hierarchy (h1, h2, h3, etc.)
• Use <button> for clickable actions
• Use <a> for navigation
• Use <form> for form submissions
• Use semantic HTML5 tags (header, nav, main, footer, section, article)

11.4. ARIA LABELS
-----------------
• Add aria-label for icon-only buttons
• Use aria-describedby for form hints
• Use aria-invalid for form errors
• Use role attributes where appropriate

11.5. KEYBOARD NAVIGATION
-------------------------
• All interactive elements must be keyboard accessible
• Tab order must be logical
• Modal traps focus within itself
• Escape key closes modals and dropdowns

14. PERFORMANCE GUIDELINES - HƯỚNG DẪN HIỆU SUẤT
================================================================================

14.1. IMAGE OPTIMIZATION
------------------------
• Use WebP format where possible
• Compress images before upload
• Use responsive images (srcset)
• Lazy load images below fold
• Provide width and height attributes

14.2. CSS OPTIMIZATION
----------------------
• Use Tailwind's purge feature
• Avoid inline styles
• Minimize custom CSS
• Use utility classes
• Group repeated patterns into components

14.3. JAVASCRIPT OPTIMIZATION
------------------------------
• Code splitting
• Lazy load components
• Minimize re-renders
• Use React.memo for expensive components
• Debounce search inputs

================================================================================
15. BEST PRACTICES - THỰC HÀNH TỐT NHẤT
================================================================================

15.1. CONSISTENCY
-----------------
• Use the same component patterns throughout
• Maintain consistent spacing
• Follow naming conventions
• Reuse existing components before creating new ones

15.2. MAINTAINABILITY
---------------------
• Keep components small and focused
• Use composition over inheritance
• Document complex logic
• Write descriptive variable names
• Avoid magic numbers (use constants)

15.3. CODE ORGANIZATION
-----------------------
components/
  ├── ui/           # Reusable UI components
  ├── admin/        # Admin-specific components
  ├── Layout.jsx    # Layout wrapper
  └── Navbar.jsx    # Navigation

pages/              # Page components
services/           # API and business logic
utils/              # Utility functions
contexts/           # React contexts

15.4. TESTING
-------------
• Test on multiple browsers (Chrome, Firefox, Safari)
• Test on multiple devices (mobile, tablet, desktop)
• Test keyboard navigation
• Test with screen readers
• Test form validation
• Test error states
